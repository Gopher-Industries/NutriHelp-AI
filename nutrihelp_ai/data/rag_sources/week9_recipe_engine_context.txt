The recipe engine does not work directly with raw images or audio. Instead, it relies on structured context generated by other system components, such as the image classifier and user input modules. This makes the system easier to maintain and helps reduce errors when generating recipes.

From the image classification side, the recipe engine receives detected food items from one or multiple images, confidence scores for each detected item, and an unclear image flag when confidence is too low. Only ingredients with sufficient confidence are treated as reliable inputs. Items with low confidence are either ignored or treated as optional, depending on the situation.

From the user side, the engine also considers dietary preferences such as vegetarian, low-carb, or halal, allergies or food restrictions, cuisine preferences, and optional calorie or portion requirements.

By combining these inputs, the recipe engine works with clean and meaningful context instead of raw data. This allows it to generate recipes that are more accurate, safer, and better aligned with user expectations.

Using structured AI context helps the recipe engine avoid guessing ingredients, handle unclear images more safely, adapt recipes to user preferences, and keep the system modular and easier to extend later.
